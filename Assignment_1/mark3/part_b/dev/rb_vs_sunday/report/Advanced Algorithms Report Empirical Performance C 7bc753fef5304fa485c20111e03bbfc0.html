Advanced Algorithms Report: Empirical Performance Comparison

1. Introduction

This report presents an empirical comparison of several string matching algorithms: Binary Sunday, Gusfield Z, Knuth-Morris-Pratt (KMP), and Rabin-Karp. The goal is to demonstrate specific situations where:

  1. Binary Sunday is at least twice as fast as Gusfield Z.
  1. KMP is at least twice as fast as Rabin-Karp.
  1. Rabin-Karp is at least twice as fast as Sunday.

2. Algorithms Description

2.1 Binary Sunday Algorithm

The Binary Sunday algorithm shifts the search window based on the characters in the pattern using a shift table. It's known for its efficient handling of mismatches by jumping large distances, thus reducing the number of comparisons.

2.2 Gusfield Z Algorithm

The Gusfield Z algorithm builds a Z-array for the concatenation of the pattern and the text, which helps to find matches efficiently. This algorithm is particularly effective in scenarios with specific text structures.

2.3 Knuth-Morris-Pratt (KMP) Algorithm

KMP preprocesses the pattern to create a longest prefix suffix (LPS) array, which allows the search phase to skip unnecessary comparisons, resulting in improved performance for certain types of text.

2.4 Rabin-Karp Algorithm

Rabin-Karp uses a hash function to compare the hash values of the pattern and the text's substrings. It is particularly effective for multiple pattern matching and can be implemented simply. Its average performance is good for hashable patterns, but it can suffer from hash collisions.

3. Benchmarking Setup

The benchmarking was conducted using randomly generated text of 100,000 characters and patterns of 50 characters. The performance was measured by the time taken to find the pattern in the text. The test cases aimed to ensure reproducibility and were designed to highlight the strengths and weaknesses of each algorithm.

class StringSearchAlgorithms:
    @staticmethod
    def sunday_search(text, pattern):
        m, n = len(pattern), len(text)
        if m > n: return -1
        shift = {pattern[i]: m - i for i in range(m)}
        i = 0
        while i <= n - m:
            if text[i:i + m] == pattern:
                return i
            i += shift.get(text[i + m], m)
        return -1

    @staticmethod
    def gusfield_z_search(text, pattern):
        concat = pattern + "$" + text
        Z = [0] * len(concat)
        l, r, K = 0, 0, 0
        for i in range(1, len(concat)):
            if i > r:
                l, r = i, i
                while r < len(concat) and concat[r] == concat[r - l]:
                    r += 1
                Z[i] = r - l
                r -= 1
            else:
                K = i - l
                if Z[K] < r - i + 1:
                    Z[i] = Z[K]
                else:
                    l = i
                    while r < len(concat) and concat[r] == concat[r - l]:
                        r += 1
                    Z[i] = r - l
                    r -= 1
        for i in range(len(Z)):
            if Z[i] == len(pattern):
                return i - len(pattern) - 1
        return -1

    @staticmethod
    def kmp_search(text, pattern):
        def compute_lps(pattern, m, lps):
            length = 0
            i = 1
            while i < m:
                if pattern[i] == pattern[length]:
                    length += 1
                    lps[i] = length
                    i += 1
                else:
                    if length != 0:
                        length = lps[length - 1]
                    else:
                        lps[i] = 0
                        i += 1

        m, n = len(pattern), len(text)
        lps = [0] * m
        j = 0
        compute_lps(pattern, m, lps)
        i = 0
        while i < n:
            if pattern[j] == text[i]:
                i += 1
                j += 1
            if j == m:
                return i - j
            elif i < n and pattern[j] != text[i]:
                if j != 0:
                    j = lps[j - 1]
                else:
                    i += 1
        return -1

    @staticmethod
    def rabin_karp_search(text, pattern, q=101):
        d = 256
        m, n = len(pattern), len(text)
        p, t, h = 0, 0, 1
        for i in range(m - 1):
            h = (h * d) % q
        for i in range(m):
            p = (d * p + ord(pattern[i])) % q
            t = (d * t + ord(text[i])) % q
        for i in range(n - m + 1):
            if p == t:
                if text[i:i + m] == pattern:
                    return i
            if i < n - m:
                t = (d * (t - ord(text[i]) * h) + ord(text[i + m])) % q
                if t < 0:
                    t += q
        return -1

class PerformanceTester:
    def __init__(self, text_length, pattern_length):
        self.text = self.generate_text(text_length)
        self.pattern_length = pattern_length

    @staticmethod
    def generate_text(length):
        return ''.join(random.choices(string.ascii_lowercase + string.whitespace, k=length))

    @staticmethod
    def generate_pattern(length):
        return ''.join(random.choices(string.ascii_lowercase + string.whitespace, k=length))

    @staticmethod
    def measure_performance(text, pattern, search_func):
        start_time = time.time()
        search_func(text, pattern)
        return time.time() - start_time

    def run_tests(self):
        pattern_sunday_vs_gusfield = self.generate_pattern(self.pattern_length)
        pattern_kmp_vs_rk = self.generate_pattern(self.pattern_length)
        pattern_rk_vs_sunday = self.generate_pattern(self.pattern_length)

        results = {
            'Sunday vs Gusfield Z': (
                self.measure_performance(self.text, pattern_sunday_vs_gusfield, StringSearchAlgorithms.sunday_search),
                self.measure_performance(self.text, pattern_sunday_vs_gusfield, StringSearchAlgorithms.gusfield_z_search)
            ),
            'KMP vs Rabin-Karp': (
                self.measure_performance(self.text, pattern_kmp_vs_rk, StringSearchAlgorithms.kmp_search),
                self.measure_performance(self.text, pattern_kmp_vs_rk, StringSearchAlgorithms.rabin_karp_search)
            ),
            'Rabin-Karp vs Sunday': (
                self.measure_performance(self.text, pattern_rk_vs_sunday, StringSearchAlgorithms.rabin_karp_search),
                self.measure_performance(self.text, pattern_rk_vs_sunday, StringSearchAlgorithms.sunday_search)
            )
        }

        return results

class Plotter:
    @staticmethod
    def plot_results(results):
        fig = go.Figure()

        for comparison, times in results.items():
            algs, times = zip(*[(alg, time) for alg, time in zip(comparison.split(' vs '), times)])
            fig.add_trace(go.Bar(
                x=list(algs),
                y=list(times),
                name=comparison
            ))

        fig.update_layout(
            title='Algorithm Performance Comparison',
            xaxis=dict(title='Algorithms'),
            yaxis=dict(title='Time (seconds)'),
            barmode='group',
            bargap=0.15,
            bargroupgap=0.1
        )

        current_file = Path(__file__).resolve()
        with open(current_file.parent / 'part_b.html', 'w') as f:
            f.write(fig.to_html())

5. Analysis

5.1 Sunday vs Gusfield Z

The Sunday algorithm performed significantly faster than the Gusfield Z algorithm, confirming the hypothesis that Sunday can be at least twice as fast in some cases.

5.2 KMP vs Rabin-Karp

The KMP algorithm outperformed the Rabin-Karp algorithm, demonstrating that KMP can be at least twice as fast under certain conditions.

5.3 Rabin-Karp vs Sunday

The Rabin-Karp algorithm was generally slower than the Sunday algorithm in our tests. However, we conducted an extensive search to find cases where Rabin-Karp is significantly faster. By generating specific test conditions, we were able to identify such cases.

5.3.1 Efficient Multiple Pattern Matching

Rabin-Karp is effective for searching multiple patterns simultaneously, using a single hash function to compare hash values.

5.3.2 Simplicity in Implementation

The algorithm is relatively straightforward to implement for single pattern searches.

5.3.3 Good Average Performance

Rabin-Karp performs well on average when the hash function distributes hash values uniformly and minimizes collisions.

def generate_rabin_karp_test_text(pattern_length, text_length, pattern_frequency):
    large_alphabet = string.ascii_letters + string.digits + string.punctuation
    pattern = ''.join(random.choices(large_alphabet, k=pattern_length))
    text = ''.join(random.choices(large_alphabet, k=text_length))
    positions = sorted(random.sample(range(text_length - pattern_length), pattern_frequency))
    for pos in positions:
        text = text[:pos] + pattern + text[pos + pattern_length:]
    for _ in range(pattern_frequency // 2):
        collision_seq = list(pattern)
        random.shuffle(collision_seq)
        collision_seq = ''.join(collision_seq)
        pos = random.randint(0, text_length - pattern_length)
        text = text[:pos] + collision_seq + text[pos + pattern_length:]
    return text, pattern
